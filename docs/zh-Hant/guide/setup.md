# 權限設定

IconChanger 需要管理員權限才能更改應用程式圖示。首次啟動時，應用程式會提供自動設定。

## 自動設定（推薦）

1. 啟動 IconChanger。
2. 在提示時點擊 **設定** 按鈕。
3. 輸入管理員密碼。

應用程式會在 `~/.iconchanger/helper.sh` 建立輔助腳本，並設定 sudoers 規則使其無需每次輸入密碼。

## 手動設定

如果自動設定失敗，可以手動設定：

1. 打開終端機。
2. 執行：

```bash
sudo visudo
```

3. 在末尾加入：

```
ALL ALL=(ALL) NOPASSWD: /Users/<你的使用者名稱>/.iconchanger/helper.sh
```

將 `<你的使用者名稱>` 替換為你的 macOS 使用者名稱。

## 驗證設定

設定完成後，應用程式應該在側邊欄顯示應用程式列表。如果仍然看到設定提示，說明設定可能未正確套用。

可以從選單列的 **...** 選單中選擇 **檢查設定狀態** 來驗證。

## 限制

受 macOS 系統完整性保護 (SIP) 限制，無法更改系統應用程式的圖示。這是 macOS 的限制，無法繞過。

常見的受 SIP 保護的應用程式包括：
- Finder（訪達）
- Safari（部分 macOS 版本）
- `/System/Applications/` 中的其他系統應用程式
