# Αρχική ρύθμιση

Το IconChanger χρειάζεται δικαιώματα διαχειριστή για να αλλάζει εικονίδια εφαρμογών. Κατά την πρώτη εκκίνηση, η εφαρμογή προσφέρει να το ρυθμίσει αυτόματα.

## Αυτόματη ρύθμιση (Συνιστάται)

1. Εκκινήστε το IconChanger.
2. Κάντε κλικ στο κουμπί **Setup** όταν σας ζητηθεί.
3. Εισαγάγετε τον κωδικό διαχειριστή σας.

Η εφαρμογή θα δημιουργήσει ένα βοηθητικό σενάριο στη διαδρομή `~/.iconchanger/helper.sh` και θα ρυθμίσει έναν κανόνα sudoers ώστε να εκτελείται χωρίς να ζητά κωδικό κάθε φορά.

## Χειροκίνητη ρύθμιση

Αν η αυτόματη ρύθμιση αποτύχει, μπορείτε να τη ρυθμίσετε χειροκίνητα:

1. Ανοίξτε το Terminal.
2. Εκτελέστε:

```bash
sudo visudo
```

3. Προσθέστε την ακόλουθη γραμμή στο τέλος:

```
ALL ALL=(ALL) NOPASSWD: /Users/<your-username>/.iconchanger/helper.sh
```

Αντικαταστήστε το `<your-username>` με το πραγματικό σας όνομα χρήστη macOS.

## Επαλήθευση ρύθμισης

Μετά τη ρύθμιση, η εφαρμογή θα πρέπει να εμφανίζει τη λίστα εφαρμογών στην πλαϊνή μπάρα. Αν εμφανιστεί ξανά η οθόνη ρύθμισης, η διαμόρφωση ενδέχεται να μην εφαρμόστηκε σωστά.

Μπορείτε να επαληθεύσετε τη ρύθμιση από τη γραμμή μενού: κάντε κλικ στο μενού **...** και επιλέξτε **Check Setup Status**.

## Περιορισμοί

Εφαρμογές που προστατεύονται από το System Integrity Protection (SIP) του macOS δεν μπορούν να αλλάξουν εικονίδιο. Αυτός είναι περιορισμός του macOS και δεν μπορεί να παρακαμφθεί.

Συνήθεις εφαρμογές που προστατεύονται από το SIP:
- Finder
- Safari (σε ορισμένες εκδόσεις macOS)
- Άλλες εφαρμογές συστήματος στη διαδρομή `/System/Applications/`
